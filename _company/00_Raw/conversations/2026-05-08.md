### 3. 슬리피지 최소화 로직 (`slippage_estimator.py`)
슬리피지는 주문을 넣는 시점의 예상 가격과 실제 체결되는 가격의 차이입니다. 이를 예측하는 것이 핵심입니다. `Order Book` 데이터를 활용해야 합니다.

$$ \text{Estimated Price} = P_{mid} - (S_{order} / V_{liquidity}) $$
*   $P_{mid}$: 현재 Mid-Price (최우선 호가 평균).
*   $S_{order}$: 우리가 주문하려는 금액/수량.
*   $V_{liquidity}$: 해당 가격대(호가)의 유동성 깊이 (Order Book Depth, 매도/매수 누적 물량 합계).

```python
def estimate_slippage(symbol: str, order_quantity: float, order_book_data: dict) -> tuple[float, float]:
    """
    주어진 수량으로 주문 시 예상되는 슬리피지 가격을 계산합니다.
    Args: 
        order_book_data: 현재 Order Book (Bid/Ask 리스트 포함).
    Returns: (Estimated Price, Estimated Slippage %)
    """
    # [1] 최적의 호가 깊이 찾기 (가장 적은 슬리피지를 유발하는 가격대)
    best_ask_price = order_book_data['asks'][0]['price']
    available_liquidity = 0.0

    # [2] 주문 수량 대비 예상되는 시장 충격을 계산합니다.
    # (간단화 모델: Order Book의 누적 깊이를 기준으로 선형 하락을 가정)
    if order_quantity > 1: # 최소 거래 단위 확인
        # 유동성이 충분하다면, 평균 가격에서 작은 오차율만 반영합니다.
        estimated_price = best_ask_price * (1 + 0.0001) 
        slippage_percent = 0.05  # 예시 슬리피지 (%)
    else:
        # 거래량이 적으면 시장 가격과 거의 동일하다고 가정합니다.
        estimated_price = best_ask_price
        slippage_percent = 0.0

    return estimated_price, slippage_percent


# 사용법: price, slippage = estimate_slippage('XRPUSDT', 1000.0, current_order_book)
```

### 🔑 Step 1: API 키 안전하게 환경 변수로 설정하기 (필수)

**핵심 원칙:** API Key와 Secret Key는 절대 코드 내부에 직접 하드코딩해서는 안 됩니다. 운영체제(OS)의 환경 변수를 사용해야 합니다.

#### A. Linux / macOS 사용자 가이드 (Bash/Zsh 기준)
터미널에 아래 명령어를 실행하여 키를 설정합니다. 이 명령어는 현재 세션에서만 유효하며, 영구적으로 적용하려면 `~/.bashrc` 또는 `~/.zshrc` 파일의 맨 끝에 추가해야 합니다.

```bash
# 1. 환경 변수 설정 (예시 값으로 대체하세요)
export BINANCE_API_KEY="YOUR_ACTUAL_API_KEY"
export BINANCE_SECRET_KEY="YOUR_ACTUAL_SECRET_KEY"

# 2. 영구 적용을 위해 .bashrc 또는 .zshrc에 추가 후 Source 명령 실행 (선택 사항)
echo 'export BINANCE_API_KEY="..."' >> ~/.zshrc
echo 'export BINANCE_SECRET_KEY="..."' >> ~/.zshrc
source ~/.zshrc # 변경된 설정을 즉시 적용
```

#### B. Windows 사용자 가이드 (PowerShell 기준)
Windows의 PowerShell을 사용하는 것이 가장 권장됩니다.

```powershell
# 1. 환경 변수 설정 (예시 값으로 대체하세요)
$env:BINANCE_API_KEY="YOUR_ACTUAL_API_KEY"
$env:BINANCE_SECRET_KEY="YOUR_ACTUAL_SECRET_KEY"

# 참고: 시스템 전체에 영구 적용하려면 Windows 검색창에서 '환경 변수 편집'을 통해 GUI로 설정하는 것을 추천합니다.
```

### 3. 핵심 아키텍처 상세 설계 (The Blueprint)

#### A. 데이터 파이프라인 개선: ETL/ELT 구조
데이터 수집은 세 가지 스트림으로 분리하여 관리합니다. 모든 데이터는 **Staging Area**를 거쳐 정제된 후, 적절한 저장소로 이동합니다.

1.  **Market Data Stream (구조화):**
    *   `Source`: Binance API (`asyncio` 이용), Yahoo Finance 등.
    *   `Process`: Time-Series Aggregator Module (Pandas 기반). 슬리피지/수수료를 포함한 정밀 계산 로직 적용.
    *   `Target`: **Time-Series Database** (InfluxDB 또는 PostgreSQL + TimescaleDB) — 시계열 데이터에 최적화.
2.  **Knowledge Data Stream (비정형):**
    *   `Source`: Web Scrapers, News APIs (RSS), 법률/규제 문서 API.
    *   `Process`: Document Parser Module $\rightarrow$ Chunking $\rightarrow$ Embedding Model 호출.
    *   `Target`: **Vector Database**. (RWA 분석 및 에이전트의 근거 자료로 활용).

#### B. Multi-Agent Core 설계: Strategy Simulator 구현
핵심 로직은 'Orchestrator'가 주도하며, 각 Agent는 독립적인 `Tool Library`를 통해 기능을 수행합니다.

1.  **입력:** 사용자 요청 (예: "XRP에 대한 분기별 리스크 조정 수익률 시뮬레이션").
2.  **Orchestrator (LangChain):**
    *   요청을 분석하여 필요한 **Agent들**과 그들의 실행 순서를 결정합니다. (계획 단계)
3.  **Agent 1: Market Analyst Agent:**
    *   `Tool Call`: `get_recent_trends(asset)` $\rightarrow$ Time-Series DB 호출.
    *   `Tool Call`: `search_vector_db("XRP 규제 변화")` $\rightarrow$ Vector DB 질의 (최신 정보를 근거로 삼음).
    *   **Output:** 시장 분석 보고서 초안.
4.  **Agent 2: Risk Manager Agent:**
    *   `Tool Call`: `calculate_drawdown(strategy, period)` $\rightarrow$ 백테스팅 엔진 호출.
    *   `Input`: Market Analyst Agent의 출력을 받아 검증하고 위험 요소 점수화.
    *   **Output:** 리스크 경고 및 수정 제안.
5.  **최종 출력:** Orchestrator가 모든 Agent의 결과를 취합하여, 인간이 이해하기 쉬운 '전략 보고서' 형태로 포맷팅합니다.

### 4. 필수 개발 스크립트 예시 (Actionable Code Snippet)

다음은 환경변수 및 비동기 처리를 고려한 Market Data 수집 초기 구조입니다.

```python
# filename: data_pipeline/market_data_collector.py

import os
import asyncio
from binance.client import Client
from datetime import datetime, timedelta

# ❗ 중요: API 키는 환경 변수에서 로드합니다.
API_KEY = os.getenv("BINANCE_API_KEY")
SECRET_KEY = os.getenv("BINANCE_SECRET_KEY")

async def fetch_historical_data(symbol: str, interval: str, limit: int) -> list:
    """
    비동기적으로 지정된 심볼의 OHLCV 데이터를 가져오는 함수.
    Rate Limit 처리를 위해 asyncio를 사용합니다.
    """
    if not API_KEY or not SECRET_KEY:
        raise EnvironmentError("Binance API Key/Secret이 환경 변수에 설정되지 않았습니다.")

    client = Client(API_KEY, SECRET_KEY)
    print(f"[{datetime.now().strftime('%H:%M')}] {symbol} 데이터 요청 시작...")

    try:
        # 실제로는 비동기 라이브러리를 사용하여 Rate Limit을 관리해야 합니다.
        klines = client.get_historical_klines(symbol, interval, f"{limit} period ago UTC")
        
        # 데이터 파싱 및 반환 로직 (실제 DB에 저장하는 로직 추가 필요)
        data = []
        for kline in klines:
            timestamp = kline[0] / 1000 # ms -> s
            open_price = float(kline[1])
            close_price = float(kline[4])
            high = float(kline[2])
            low = float(kline[3])
            volume = float(kline[5])
            data.append({
                'timestamp': timestamp, 
                'open': open_price, 
                'close': close_price, 
                'high': high, 
                'low': low, 
                'volume': volume
            })
        return data

    except Exception as e:
        print(f"🚨 데이터 수집 중 에러 발생 ({symbol}): {e}")
        # 로깅 시스템 (예: Sentry 또는 전용 로그 DB)에 기록하는 코드를 추가해야 합니다.
        return []

async def run_collector():
    """주요 실행 함수."""
    symbols = ["XRPUSDT", "BTCUSDT"]
    tasks = [fetch_historical_data(s, "1h", 500) for s in symbols]
    # 모든 태스크를 동시에 비동기 실행 (병렬 처리)
    results = await asyncio.gather(*tasks) 

    for i, data in enumerate(results):
        print(f"\n✅ {symbols[i]} 데이터 수집 완료: {len(data)}개 레코드.")
        # 다음 단계: 결과를 DB에 저장하는 로직 호출 (save_to_timeseries_db(data))

if __name__ == "__main__":
    asyncio.run(run_collector())
```

## 🔬 Use Case별 상세 기술 요소 및 기회 분석 (Deep Dive)

### 1. 예측 기반 설비 진단 (PdM, Predictive Maintenance)
*   **필요 기술 요소:**
    *   **센싱:** 가속도계(Accelerometer), 온도 센서, 음향/진동 센서 등 물리적 신호 측정 장치.
    *   **네트워크:** MQTT 기반의 저전력 광역 네트워크 (LPWAN) 및 엣지 컴퓨팅 게이트웨이.
    *   **ML 모델:** **LSTM (Long Short-Term Memory)** 또는 **Transformer Model**을 이용한 시계열 데이터 분석(Time-Series Analysis), 이상 탐지(Anomaly Detection).
*   **주요 산업 적용 사례:** 발전소 터빈, 공장 컨베이어 벨트 모터, 대형 펌프.
*   **💡 상업적 기회 요인 (Opportunity Gap):** 현재의 PdM은 '고장 발생 가능성' 예측에 머무르는 경우가 많습니다. **진정한 기회는 '고장이 발생할 원인(Root Cause)'을 센서 데이터만으로 역추적하여 엔지니어에게 제시하는 진단 보고 시스템 구축**입니다.

### 2. 원격 생체 신호 모니터링 (RPM, Remote Patient Monitoring)
*   **필요 기술 요소:**
    *   **센싱:** ECG/PPG 센서(심전도), 혈당 측정 센서, 동작 감지 가속도계 등 비접촉 및 웨어러블 기기.
    *   **네트워크:** HIPAA 준수(Compliance)를 위한 암호화된 통신 채널 (VPN).
    *   **ML 모델:** **Multi-modal Fusion Model**을 사용하여 여러 센서의 데이터를 통합하고, 미세한 생체 신호 변화 패턴을 인식하는 분류 및 회귀 알고리즘.
*   **주요 산업 적용 사례:** 만성질환 관리 센터, 군 병원, 가정 돌봄 서비스.
*   **💡 상업적 기회 요인 (Opportunity Gap):** 데이터가 개인에게만 국한됩니다. **다수의 환자 데이터를 익명화하여 묶고(Cohort Analysis), 특정 질병군이 공통적으로 겪는 '미진단 위험 신호'를 예측하는 집단 지능형 모델 개발**에 초점을 맞출 수 있습니다.

### 3. 지능형 스마트 시티 관리 (Smart City)
*   **필요 기술 요소:**
    *   **센싱:** 고해상도 카메라(CCTV), 미세먼지/가스 센서, IoT 주차 센서 등 다종 센서 네트워크.
    *   **네트워크:** 5G-MEC (Mobile Edge Computing)를 활용한 초저지연 데이터 처리 인프라.
    *   **ML 모델:** **객체 인식(Object Detection)** 및 **행동 패턴 분석(Behavioral Analysis)**을 위한 Computer Vision 기술, 시계열 예측 모델.
*   **주요 산업 적용 사례:** 교통 신호 제어, 재난 발생 지점 감지, 에너지 수요 관리 시스템 (EMS).
*   **💡 상업적 기회 요인 (Opportunity Gap):** 개별 센서 데이터 분석은 많지만, **'이종 센서 간의 상관관계'를 이용한 종합적인 의사결정 지원 플랫폼(Decision Support System)** 구축입니다. 예: "교통 체증 + 미세먼지 레벨 증가 = 대중교통 분산 및 특정 구간 차량 통제 필요"와 같은 복합적 시나리오 제시.

### 4. 정밀 농업/작물 관리 (Precision Agriculture)
*   **필요 기술 요소:**
    *   **센싱:** 토양 pH, 수분 함량, 질소/인산염 센서, 드론 기반 Multispectral 카메라(식생 지수 분석).
    *   **네트워크:** LoRaWAN 또는 전용 무선 통신망.
    *   **ML 모델:** **CNN (Convolutional Neural Network)**을 이용한 식물 이미지 진단 및 병해충 인식, 강화 학습을 통한 최적 자원 분배 알고리즘.
*   **주요 산업 적용 사례:** 대규모 밭 농가, 시설 원예 농장.
*   **💡 상업적 기회 요인 (Opportunity Gap):** 현재는 '비료를 뿌리는 시점'에 초점이 맞춰져 있습니다. **작물 생체 데이터(Chlorophyll Index 등)와 기상 예측을 결합하여, "다음 48시간 내 특정 영양소가 부족해질 확률"과 같이 시간 기반의 미세한 처방**이 필요합니다.

### 5. 스마트 리테일/재고 관리 (Smart Retail)
*   **필요 기술 요소:**
    *   **센싱:** LoRa/UWB(초광대역), 비전 센서, 무게 감지 센서(Load Cell).
    *   **네트워크:** 내부 Wi-Fi 6 기반의 고밀도 데이터 수집망.
    *   **ML 모델:** **GAN (Generative Adversarial Network)**을 활용한 가상 고객 시뮬레이션 및 상품 배치 최적화, 행동 패턴 클러스터링 알고리즘.
*   **주요 산업 적용 사례:** 백화점, 대형 마트, 물류 창고.
*   **💡 상업적 기회 요인 (Opportunity Gap):** 재고 관리 시스템은 이미 많이 도입되었습니다. 핵심 격차는 **'데이터 기반의 인력 최적 배치'입니다.** 예: "오늘 오후 2시~4시는 A 코너에 특정 상품이 몰릴 확률이 높으니, 해당 시간대에는 직원 B가 배치되어야 한다."와 같이 노동 리소스를 AI로 관리하는 시스템.

### 6. 자율 이동체 지원 시스템 (AV Support)
*   **필요 기술 요소:**
    *   **센싱:** LiDAR, Radar, 카메라(Camera), 초음파 센서 등 다중 센서 퓨전.
    *   **네트워크:** 차량 내부 고속 통신망 (CAN Bus, Ethernet).
    *   **ML 모델:** **BEV (Bird's Eye View) 기반의 실시간 상황 인지 및 예측(Prediction)** 알고리즘, 강화 학습을 통한 최적 제어 경로 탐색.
*   **주요 산업 적용 사례:** 자율 배송 로봇, 무인 물류 창고 운반 차량.
*   **💡 상업적 기회 요인 (Opportunity Gap):** 자율주행 기술 자체는 거대 기업들이 주도합니다. 우리의 틈새 시장은 **'특정 환경에 최적화된 보조 모듈 개발'**입니다. 예를 들어, '좁고 복잡한 실내 아파트 단지 내 배송 경로의 돌발 장애물 예측 및 우회 로직'처럼 특정 공간 제약 조건에 특화된 솔루션이 필요합니다.

## 🌐 Use Case별 표준화된 프로젝트 기획 아웃라인

### 1. 예측 기반 설비 진단 (Predictive Maintenance, PdM)
*   **산업:** 스마트 팩토리, 에너지 플랜트
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |
| :---: | :--- | :--- |
| **(1) 문제 정의** | 설비 고장이 발생한 후 수리하는 사후 대응(Reactive Maintenance) 방식은 비효율적이며, 예상치 못한 다운타임으로 막대한 생산 손실을 초래함. | *Pain Point:* 갑작스러운 설비 오작동 및 예측 불가한 가동 중단 시간(Downtime). |
| **(2) 아키텍처 설계** | ① **[IoT Edge]**: 진동, 온도, 전류 센서 → ② **[데이터 전송]**: MQTT 프로토콜을 통한 데이터 수집 → ③ **[AI Cloud]**: 시계열 데이터 분석 및 이상 징후 패턴 인식 (LSTM/Transformer 모델) → ④ **[액션 레이어]**: 고장 예측 시간(Time-to-Failure, TTF) 도출 및 유지보수팀 알림. | *핵심:* 실시간 센서 데이터 수집, 시계열 AI 분석, 경고 시스템 연동. |
| **(3) KPI 측정** | - **설비 가용률 (Availability Rate)**: ⬆️ 목표치 설정 (예: 5% 향상)<br>- **평균 고장 간격 (MTBF, Mean Time Between Failures)**: ⬆️ 증가<br>- **사전 예방 정비 비율**: 📈 측정 및 보고. | *측정 기준:* 유지보수 비용 절감액(Cost Saving) 산출. |

### 2. 원격 생체 신호 모니터링 (Remote Patient Monitoring, RPM)
*   **산업:** 헬스케어, 요양 시설
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |
| :---: | :--- | :--- |
| **(1) 문제 정의** | 만성 질환자가 일상생활 중 위험 징후를 발견하기 어려우며, 병원에 상주할 경우 비용과 거주 환경 제약이 따름. 실시간 모니터링 공백 발생. | *Pain Point:* 취약 시간대 감지 실패 및 의료진의 광범위한 인력 투입 필요. |
| **(2) 아키텍처 설계** | ① **[IoT Edge]**: 웨어러블 센서 (심박, 체온, 혈당 등) → ② **[데이터 전송]**: 암호화된 무선 통신 채널(5G/LTE-M)을 통한 데이터 스트리밍 → ③ **[AI Cloud]**: 정상 범위를 벗어나는 패턴 변화 감지 및 복합 위험 지수 산출 (Anomaly Detection Model). ④ **[액션 레이어]**: 의료진의 모바일 앱으로 즉각적인 경고 알림 및 원격 상담 시스템 연동. | *핵심:* 생체 데이터 통합 분석, 위기 상황 분류 알고리즘, 보안(HIPAA 등) 준수 설계. |
| **(3) KPI 측정** | - **위험 감지 시간 (Detection Time)**: ⬇️ 단축 목표<br>- **응급상황 대응율**: ⬆️ 향상된 비율 확보<br>- **원격 모니터링 만족도**: 설문 및 사용자 인터뷰 기반 정량화. | *측정 기준:* 응급실 방문 감소율(Cost Saving) 산출. |

### 3. 지능형 스마트 시티 관리 (Smart City Management)
*   **산업:** 도시 인프라, 공공 안전
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |
| :---: | :--- | :--- |
| **(1) 문제 정의** | 교통 흐름 분석이 단편적이며(교통 카드만 사용), 재난 발생 시 다중 센서 데이터 간의 통합적인 위험 예측 시스템 부재. 효율성 및 안전성이 분산되어 있음. | *Pain Point:* 이기종 데이터(CCTV, 환경, 교통량) 결합 분석 부족 및 실시간 자원 배분 실패. |
| **(2) 아키텍처 설계** | ① **[IoT Edge]**: CCTV (비전), 미세먼지 센서, 지능형 가로등 등 분산 센싱 네트워크 → ② **[데이터 전송]**: 게이트웨이에서 데이터 정규화 및 필터링 → ③ **[AI Cloud]**: Multi-modal 데이터 융합 분석(Data Fusion)을 통한 위험도 매핑 (Graph Neural Network 활용 가능). ④ **[액션 레이어]**: 신호등 제어 시스템 연동, 재난 발생 지역에 대한 자동 자원 배분 지시. | *핵심:* 대규모 이기종 데이터 처리 능력, 실시간 트래픽/위험 예측 모델, 중앙 관제 플랫폼 구축. |
| **(3) KPI 측정** | - **교통 흐름 최적화율**: ⬆️ 증가 (예: 평균 통행 시간 15% 단축)<br>- **재난 인지 및 대응 시간**: ⬇️ 감소<br>- **에너지 사용 효율성**: 📈 개선 비율(스마트 그리드 연동 시). | *측정 기준:* 운영 비용 절감액(운전 인력, 에너지), 사고율 감소. |

### 4. 정밀 농업/작물 관리 (Precision Agriculture)
*   **산업:** 농업, 식량 안보
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |
| :---: | :--- | :--- |
| **(1) 문제 정의** | 농작물 재배에 필요한 자원(물, 비료, 방제 약품 등)을 일괄적으로 투입하여 낭비가 심하고, 병충해 발생 시기에 대한 조기 경보 시스템이 부족함. | *Pain Point:* 과도한 자원 사용으로 인한 경제적 손실 및 환경 오염 유발. |
| **(2) 아키텍처 설계** | ① **[IoT Edge]**: 토양 센서 (습도, pH), 드론/위성 이미지 (멀티 스펙트럼 카메라) → ② **[데이터 전송]**: 필드 게이트웨이를 통한 데이터 취합 및 암호화 전송 → ③ **[AI Cloud]**: 작물의 성장 단계별 건강 진단 및 최적 투입량 예측 (Computer Vision, CNN 기반 이미지 분석). ④ **[액션 레이어]**: 로봇 또는 드론에 대한 국지적 제어 명령(Targeted Action) 전달. | *핵심:* 원격 감지 데이터 분석, 작물 생육 모델링 AI, 자원 사용 최적화 알고리즘. |
| **(3) KPI 측정** | - **자원 투입 비용 절감률**: ⬆️ (비료/물 사용량 기준)<br>- **수확량 증대율**: ⬆️ (가장 핵심적인 목표 지표)<br>- **병충해 조기 감지 정확도**: 💯 높은 예측 정확도를 확보. | *측정 기준:* 단위 면적당 순이익(Net Profit per Hectare) 증가분 산출. |

### 5. 스마트 리테일/재고 관리 (Smart Retail & Inventory)
*   **산업:** 유통, 물류창고, 오프라인 매장
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |
| :---: | :--- | :--- |
| **(1) 문제 정의** | 고객의 실제 동선과 구매 패턴을 파악하기 어려워 비효율적인 매장 레이아웃 유지. 재고 위치 추적이 수기로 이루어져 분실 및 재고 오차가 잦음. | *Pain Point:* '구매 여정 데이터'의 사각지대 존재, 노동 집약적이고 느린 재고 관리 프로세스. |
| **(2) 아키텍처 설계** | ① **[IoT Edge]**: 비전 센서 (CCTV), RFID 태그/카메라, 무게 감지 센서 → ② **[데이터 전송]**: 실시간 트래픽 및 이벤트 데이터 스트리밍 → ③ **[AI Cloud]**: 고객 동선 분석(Heatmap Generation)을 통한 구매 흐름 예측. 재고 위치 추적 및 수요 예측 (Time Series Forecasting). ④ **[액션 레이어]**: 매장 내 디지털 사이니지 제어, 자동 발주 시스템 연동. | *핵심:* 비전 AI를 활용한 행동 패턴 분석, 실시간 재고 가시성(Visibility), 수요/공급 최적화 알고리즘. |
| **(3) KPI 측정** | - **매장 체류 시간 및 동선 효율**: ⬆️ 개선<br>- **재고 오차율**: ⬇️ 감소 (목표: 0%에 근접)<br>- **판매 전환율 (Conversion Rate)**: ⬆️ 증가. | *측정 기준:* 인건비 절감액, 매출 증대 효과(Revenue Increase) 산출. |

### 6. 자율 이동 및 물류 최적화 (Autonomous Mobility & Logistics)
*   **산업:** 물류창고, 배송, 산업 현장
| 단계 | 세부 내용 및 정의 | 핵심 기술/요소 |

복잡한 계산을 UI와 분리하기 위해 핵심 로직을 별도의 함수로 구성합니다. 이는 재사용성과 테스트 용이성을 극대화합니다.

**`app.py` 파일 내부에 구현할 핵심 함수 예시:**
```python
import pandas as pd
import plotly.graph_objects as go
from typing import Tuple, List

def calculate_risk(aoc: float, lambda_val: float, max_downtime: float) -> Tuple[float, float]:
    """
    현재 위험 노출액 ($L$)과 리스크 지수를 계산합니다. (핵심 로직 1)
    L = AOC * 예상 고장 확률 * 평균 복구 지연 시간
    """
    # 예시 상수 설정: 예상 고장 확률 및 평균 복구 지연 시간을 간소화하여 사용
    estimated_failure_rate = 0.05  # 연간 예상 고장 확률 (예시)
    avg_recovery_delay = 30 # 평균 복구 지연 일수 (예시)
    
    annual_potential_loss = aoc * estimated_failure_rate * avg_recovery_delay
    risk_index = annual_potential_loss / max_downtime if max_downtime > 0 else float('inf')
    
    return annual_potential_loss, risk_index

def simulate_savings_curve(initial_cost: float, improvement_rate: float, years: int) -> pd.DataFrame:
    """
    시간 경과에 따른 절감액 시뮬레이션 데이터를 생성합니다. (핵심 로직 2)
    """
    years_data = []
    current_saving = initial_cost * improvement_rate # 초기 예상 절감액
    
    for i in range(years):
        # 매년 절감 효과가 누적되고, 시스템 도입으로 인해 효율이 증가한다고 가정
        annual_reduction = current_saving * (1 + 0.1 * (i / years)) 
        years_data.append({
            'Year': i + 1, 
            'Annual Reduction ($)': round(annual_reduction, 2)
        })
        current_saving += annual_reduction # 다음 해의 기준 절감액 증가
        
    return pd.DataFrame(years_data)

def create_roi_graph(df: pd.DataFrame) -> go.Figure:
    """
    Plotly를 사용하여 시간 경과에 따른 절감 효과 그래프를 생성합니다. (시각화 로직)
    """
    fig = go.Figure()
    fig.add_trace(go.Bar(x=df['Year'], y=df['Annual Reduction ($)'], name='예상 연간 절감액'))
    
    # 그래프 제목 및 레이아웃 설정 (C-Level 어필 강조)
    fig.update_layout(
        title="AI 도입을 통한 잠재적 손실 감소 추이", 
        xaxis_title="경과 시간 (년)", 
        yaxis_title="절감액 (단위: $)",
        template="plotly_dark" # 비즈니스 느낌 강조를 위한 다크 모드 적용 예시
    )
    return fig
```

### 🛠️ 핵심 작업: ROI 시뮬레이터 백엔드 (Mocking 구현)

아래 코드는 실제 데이터베이스나 외부 API 호출 없이도 **'손실액 기반의 투자 회수 기간(Payback Period)'** 계산이 안정적으로 작동함을 입증하는 `SimulationEngine` 클래스입니다.

**파일 이름:** `simulation_engine.py`
**설명:** 이 엔진은 입력된 Mock 데이터셋을 받아, 잠재적 손실액($L$) 대비 투자비용($C$)과 절감 효과($S$)를 계산하고 최종 ROI 지표를 도출합니다.

```python
import pandas as pd
from typing import Dict, Any, List

class SimulationEngine:
    """
    ROI 시뮬레이터의 핵심 로직 엔진. 
    외부 API 의존도를 제거하고 Mocking Data로 초기 검증을 목표로 설계됨.
    """
    def __init__(self):
        # 아키텍처적으로 데이터 소스 분리를 위한 Placeholder
        self.data_source = "MOCK" # 나중에 'LIVE_API' 또는 'DB'로 변경 가능

    def _load_data(self, source: str) -> pd.DataFrame:
        """
        데이터 로딩 계층 (Data Loading Layer). 
        현재는 Mocking Data를 강제하고, 추후 API 호출을 담당할 자리입니다.
        """
        print(f"\n[INFO] 데이터 소스 로드 시작: {source} 모드를 사용합니다.")
        if source == "MOCK":
            # --- 🟢 [Mocking Data 구현 부분] ---
            mock_data = pd.DataFrame({
                'Industry': ['중소형 조선사', '병원 시설 관리', '지역 전력 설비'],
                'Initial_CAPEX': [1500, 300, 800],  # 초기 투자 비용 (백만원)
                'Annual_Avg_Loss': [2500, 450, 900], # 연평균 손실액 (백만원)
                'Risk_Reduction_Factor': [0.7, 0.6, 0.8] # 개선 효과 계수 (0~1)
            })
            return mock_data
        # TODO: 향후 Live API/DB 연결 로직 추가 예정
        elif source == "LIVE_API":
            raise NotImplementedError("Live API 연동은 추후 환경 변수가 설정된 후 구현합니다.")
        else:
             raise ValueError(f"지원하지 않는 데이터 소스입니다: {source}")


    def run_simulation(self, data_source: str = "MOCK") -> pd.DataFrame:
        """
        시뮬레이션을 실행하고 핵심 ROI 지표를 계산합니다.
        """
        # 1. 데이터 로드 (Mocking 사용)
        try:
            df = self._load_data(data_source)
        except Exception as e:
            print(f"[ERROR] 데이터 로드 실패: {e}")
            return pd.DataFrame()

        # 2. 핵심 시뮬레이션 로직 적용 (기술적 증명 부분)
        
        # A. 예상 절감 효과 계산 (Savings = Loss * Factor)
        df['Expected_Savings'] = df['Annual_Avg_Loss'] * df['Risk_Reduction_Factor']

        # B. 투자 회수 기간(Payback Period, Years) 계산
        # Payback Period = CAPEX / Expected_Savings
        # 분모가 0일 경우 오류 방지 로직 추가
        df['Payback_Period_Years'] = df['Initial_CAPEX'] / df['Expected_Savings'].replace(0, 1e-6)

        # C. ROI 및 보험 가치 정의 (ROI는 Payback Period를 역으로 해석)
        df['Investment_Return_Rate'] = (1 - (1 / df['Payback_Period_Years'])).round(2) * 100
        
        print("\n✅ 시뮬레이션 성공: 핵심 로직이 Mocking Data로 안정적으로 작동합니다.")
        return df[['Industry', 'Initial_CAPEX', 'Expected_Savings', 'Payback_Period_Years', 'Investment_Return_Rate']]

# ======================================================
# 🚀 실행 예시 및 사용법 (테스트 코드)
# ======================================================
if __name__ == "__main__":
    engine = SimulationEngine()
    results_df = engine.run_simulation(data_source="MOCK")
    
    print("\n" + "="*60)
    print("💰 시뮬레이션 결과 (Mocking Data 기반)")
    print("="*60)
    # 보기 좋게 출력 포맷팅
    pd.set_option('display.float_format', lambda x: '%.2f' % x) 
    print(results_df)

```
